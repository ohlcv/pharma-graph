## 文档定位

这份文档不再承担下面这些职责：

- 不再重复写 frontmatter 字段规则；
- 不再重复写正文生成规范；
- 不再维护逐文件逐字段的问题清单；
- 不再用“100% / 43% / 14%”这类旧评分表示是否真正完成。

它现在只做一件事：**记录接下来改哪一批、改到哪一步、哪些内容需要回炉。**

## 当前目标

当前修订统一围绕三件事展开：

1. frontmatter 结构统一到当前 schema；
2. 正文统一按 `docs/frontmatter.md` 的新规范重写；
3. `edges_out` 需要补到可用状态，并保证 target 指向真实存在的节点。

本轮尤其要注意：

- 之前已经“写过”或“标记完成”的节点，不等于现在仍然合格；
- 凡是正文仍沿用旧写法、旧模板、旧口径的节点，都视为需要回炉；
- 凡是 `edges_out` 为空、明显缺关键关系，或 target 指向无效节点的，也都不能算完成；
- `frontmatter-audit.md` 只管记录批次和进度，不再解释具体怎么改。
