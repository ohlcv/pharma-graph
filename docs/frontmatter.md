## 一、字段参考

### type — 实体类型（决定节点形状）

| 值 | 形状 | 含义 | 示例 |
|---|---|---|---|
| `drug` | 圆角矩形 | 具体药物或制剂 | 美托洛尔、阿司匹林 |
| `disease` | 菱形 | 疾病或病理状态 | 高血压、感冒 |
| `pathogen` | 菱形 | 病原体 | 流感病毒、金黄色葡萄球菌 |
| `mechanism` | 菱形 | 作用机制或信号通路 | β受体阻滞机制 |
| `ingredient` | 六边形 | 辅料、成分、剂型 | 羟丙甲纤维素 |
| `concept` | 椭圆 | 概念、术语、章节知识点 | 生物利用度、首过效应 |
| `service` | 六边形 | 药学服务、合理用药 | 用药交代、用药咨询 |
| `pathway` | 矩形 | 信号通路、受体家族 | 肾素-血管紧张素系统 |
| `indicator` | 椭圆 | 检测指标、检验项目 | ALT、血压 |

> `disease` 和 `pathogen` 形状相同，通过 `category` 边框色区分。

### category — 系统归属（决定节点边框色）

| 值 | 边框色 | 值 | 边框色 |
|---|---|---|---|
| `cardiovascular` | 红 | `anti_infective` | 紫 |
| `respiratory` | 蓝 | `anti_tumor` | 粉 |
| `digestive` | 绿 | `blood` | 玫瑰红 |
| `endocrine` | 橙 | `immunology` | 金 |
| `musculoskeletal` | 青 | `dermatology` | 棕 |
| `antipyretic` | 浅蓝 | `nutrition` | 灰绿 |
| `anti_rheumatic` | 深橙 | `diagnostic` | 灰 |
| `anti_gout` | 紫红 | `pharmacy_practice` | 灰蓝 |

### layer — 知识层（决定节点布局分组）

| 值 | 含义 | 图中位置倾向 |
|---|---|---|
| `foundation` | 基础层：生理、病理、药理基础 | 底层/中心 |
| `system` | 药物系统层：具体药物、制剂、成分 | 中层 |
| `clinical` | 临床层：疾病、诊断、治疗方案 | 中上层 |
| `service` | 服务层：用药管理、合理用药指导 | 边缘/外围 |

> `category` = 在哪个系统（颜色）；`layer` = 在哪个层次（位置）。两者正交。

### location — 位置层级

只写实际存在的层级，字段顺序固定：

| 字段 | 来源 | 何时写 |
|---|---|---|
| `book` | 根文件名 | 必须 |
| `part` | 第一级子目录 | 有则写 |
| `chapter` | 第二级子目录或文件名 | 必须 |
| `section` | 第三级子目录 | 有子节文件才写 |
| `subsection` | 第四级子目录 | 有孙节文件才写 |
| `item` | 具体知识点名 | 有则写 |

**规则：**
- `label` 只写节点显示名称，不带“第几篇 / 第几章 / 第几节 / 第几小节”等层级序号。
- 层级序号只允许出现在 `location` 相关字段中；除 `location` 外，其他字段一律不要带“第几篇 / 第几章 / 第几节 / 第几小节”。
- `section == chapter` 时，删 `section`，只保留 `chapter`
- `section == subsection` 时，删 `subsection`
- `subsection` 只在还有第三层子节文件时才出现

### edges_out — 关系边

| 值 | 含义 | 方向示例 |
|---|---|---|
| `isa` | 是个体/实例（具体→抽象） | 美托洛尔 → β受体阻滞剂 |
| `part_of` | 属于整体 | 节 → 章，章 → 篇 |
| `mechanism` | 作用机制 | 药 → 靶点机制 |
| `causes` | 引起/导致 | 病原体 → 疾病 |
| `treats` | 治疗 | 阿司匹林 → 发热 |
| `has` | 含有/具有 | 制剂 → 辅料 |
| `relates` | 一般关联 | 同领域相关药 |

**填写规则：**

1. 先补结构归属边（`part_of`）：节→章，章→篇，知识点→所属节
2. 再补 1–3 条最关键的关联边（`relates` 或更强语义）
3. 不要为"有边而有边"；`target` 必须指向仓库里真实存在的节点 id

**各类节点最低要求：**

| 节点类型 | 最低边数 | 重点边类型 |
|---|---|---|
| 篇/章/节入口 | 1 条 `part_of` + 1–3 条下游 | `part_of`、`relates` |
| 概念类 | 1 条结构归属边 + 关键上下游 | `part_of`、`relates` |
| 具体药物类 | 1 条结构归属边 + 同剂型/机制/适应证 | `isa`、`treats`、`mechanism` |
| 制剂/工艺类 | 1 条结构归属边 + 吸收/实例关联 | `part_of`、`relates` |

**完成标准：** frontmatter 符合当前 schema + 正文五问成立 + `edges_out` 不为空且反映正文关键关系 + 所有 `target` 能在现有节点 id 中找到。

## 二、正文规范（五问法）

正文目标不是逐点解释，而是帮用户**从点到线，再到面**：搞懂当前节点 → 搞懂它和上下游的关系 → 把关系收束成整体结构。

### 五个维度

每篇正文覆盖以下五个维度，维度固定，问句可根据节点类型换写法：

| 维度 | 核心问题 | 对应 frontmatter 字段 | 回答要求 |
|---|---|---|---|
| **本体** | 这个节点到底是什么 | `label + type + summary.short` | 先定身份，说清是药/概念/机制/入口，不展开细节 |
| **意义** | 为什么要理解它 | `summary.full + layer` | 说清学习价值/理解价值/应用价值，不能只说"很重要" |
| **关联** | 它主要和什么有关 | `category + tags + edges_out` | 把 edges_out 翻译成人话，组织成有逻辑的关联网络，不要堆标签 |
| **过程** | 它在什么阶段出现 | `layer` + 流程位置 | 明确先后顺序（学习顺序/体内过程/临床流程），不要写成位置介绍 |
| **结构** | 它在整体框架里放哪里 | `location` | 交代书本层级和知识结构层级，说清是总入口/中间层/下位节点 |

### 按节点类型选问法

| 节点类型 | 本体问 | 意义问 | 关联问 | 过程问 | 结构问 |
|---|---|---|---|---|---|
| **具体药物** | 它是什么药？ | 它为什么值得单独记住？ | 它主要和什么机制、疾病或同类药有关？ | 它一般在什么用药或学习阶段出现？ | 它在整套药学知识里挂在哪个系统或章节下？ |
| **概念** | 这个概念到底在讲什么？ | 为什么这个概念要先搞懂？ | 它主要和哪些规律、现象或判断有关？ | 它通常在什么学习阶段先建立？ | 它在整套框架里属于哪一层、放在哪一块？ |
| **机制** | 这个机制讲的是哪一步怎么起作用？ | 理解这个机制能帮我看懂什么？ | 它主要和哪些药物、受体、通路或结果有关？ | 它通常发生在整个过程的哪个环节？ | 它在整条作用链或整章内容里放在哪？ |
| **疾病/状态** | 这讲的是哪种疾病、状态或结果？ | 为什么理解它很关键？ | 它和哪些病因、表现、用药或机制有关？ | 它一般出现在疾病发展的哪个阶段？ | 它在整套知识里挂在哪个系统或专题下？ |
| **制剂/工艺** | 这是什么制剂、给药途径或工艺？ | 它解决什么实际问题？ | 它和哪些剂型特点、吸收规律或使用场景有关？ | 它通常在给药流程或制剂设计的哪个环节出现？ | 它在药剂学框架里放在哪一块？ |
| **篇/章/节入口** | 这一篇/章/节主要在讲什么？ | 为什么它是这一块内容的关键入口？ | 它下面通常会展开哪些子主题？ | 它通常出现在整个学习路径的什么阶段？ | 它在整本书的大框架里处在哪个位置？ |
| **不确定时保底** | 它是什么？ | 为什么要理解它？ | 它主要和什么内容有关？ | 它通常在什么阶段出现或被使用？ | 它在整套书和知识体系里放在哪里？ |

### 从 frontmatter 判断节点类型

按优先级依次判断：

1. **先看 `type`**：`drug` → 药物类；`concept` → 概念类；`mechanism` → 机制类
2. **再看 `location`**：文件名与篇/章/节标题一致、`summary` 是总述句、`edges_out` 偏连下级主题 → 入口类（优先级高于 `type: concept`）
3. **再看 `summary`**：下定义/讲规律 → 概念/机制类；总览导读 → 入口类；强调疾病表现 → 疾病类；强调剂型工艺 → 制剂类
4. **最后看 `tags + edges_out`**：边大多连药物/适应证 → 药物类；连受体/通路 → 机制类；连下级主题 → 入口类
5. **仍模糊就用保底组合**，回答里通过语气区分

### 生成正文的执行流程

1. 判断节点类型，选对应一行问法
2. 把字段转义为人话（不要原样搬字段值）：
   - `type` → "具体药物 / 基础概念 / 机制 / 模块入口"
   - `category` → "更适合放在……系统里理解"
   - `layer` → "前面必须先建立的底层认识" / "已经更靠近临床使用"
   - `location` → "位于哪一篇哪一章，在整体结构里是什么位置"
   - `edges_out` → "它属于……" / "它通过……起作用" / "它常和……连起来看"
3. 五答各解决一个维度，互不重复；去掉任意一答，信息都应少一块
4. 把关系写出来：第三答和第五答必须有网络感，让用户知道"它和谁连着、为什么连、放在哪条线里"
5. 最后检查：没有直接复述英文枚举值；没有只罗列 tags；普通人读完后真的知道"它是什么、跟谁有关、放在哪了"

**格式要求：** 每一问显式写标题；每答 1–3 句；五答合计约 180–320 字。

### 正文示例

```markdown
# (label，注意只写节点显示名称)

## 它是什么？
它是药理学里的一个基础概念，讲的是药物作用并不只有治疗效果，还可能同时伴随不良反应。

## 它有什么用，为什么要理解它？
理解这个概念，可以帮助判断用药时为什么总要权衡疗效和风险，也能解释为什么同一种药既能治病，也可能伤害机体。

## 它主要和什么内容有关？
它和药物作用、治疗作用、不良反应、用药安全这些内容直接相关，也和后面很多合理用药判断连在一起。

## 它通常在什么阶段出现或被使用？
它通常出现在学习药物基本作用规律的早期阶段，是进入临床用药分析之前必须先建立的底层认识。

## 它在整套书和知识体系里放在哪里？
它位于药学专业知识一第二篇药理与毒理学、第四章药物对机体的作用、第一节，是后续理解药效、毒性和用药安全的重要起点。
```

## 三、批量写节点的推荐顺序

**一句话原则：先用样板定标准，再用主链建导航，最后回补通过校验但仍然"走不通"的薄弱节点。**

### 推荐顺序

1. **先做一条样板链条**：挑一条层级清楚、概念关联明显的小链（最好包含篇入口/章入口/节点/具体概念），跑通写法和边密度标准
2. **补直接上游入口**：节→章→篇→总入口，最快形成可导航主线
3. **补同链的直接下游兄弟节点**：已写`药物吸收`，就继续补`生物利用度`、`首过效应`，不跳到别处
4. **扩到相邻主链的篇/章入口**：一条链通后，再去补相邻模块入口（性价比最高）
5. **回补通过校验但关系偏薄的节点**：重点看总入口、篇/章入口、正文提到很多关联但边只有 0–1 条的节点

### 为什么这样做

- **先定样板**：统一正文口径和边密度，避免后面批量返工
- **先打主链**：图谱最早从"能看点"变成"能导航"
- **每轮小闭环**：每补完一小批就运行 `npm run validate`，扩写速度和结构可靠性并行

### 可以调整顺序的情况

- 某节点是很多边共同指向的关键枢纽 → 可优先补
- 某上游已经写好，下游某一簇明显断链 → 可先修那一簇
- 用户明确指定范围 → 以用户要求为准
