### location — 位置层级

只写实际存在的层级，字段顺序固定：

| 字段 | 来源 | 何时写 |
|---|---|---|
| `book` | 根文件名 | 必须 |
| `part` | 第一级子目录 | 有则写 |
| `chapter` | 第二级子目录或文件名 | 必须 |
| `section` | 第三级子目录 | 有子节文件才写 |
| `subsection` | 第四级子目录 | 有孙节文件才写 |
| `item` | 具体知识点名 | 有则写 |

**规则：**
- `label` 只写节点显示名称，不带"第几篇 / 第几章 / 第几节 / 第几小节"等层级序号。
- 层级序号只允许出现在 `location` 相关字段中；除 `location` 外，其他字段一律不要带"第几篇 / 第几章 / 第几节 / 第几小节"。
- `section == chapter` 时，删 `section`，只保留 `chapter`
- `section == subsection` 时，删 `subsection`
- `subsection` 只在还有第三层子节文件时才出现

### tags — 标签

用来补充 `field` 和 `category` 没有覆盖到的维度。每节点 3–8 个，不得与 `field`/`category` 重复。

**可选维度：**

| 维度 | 示例 |
|---|---|
| 机制类型 | `激动剂`, `拮抗剂`, `酶抑制剂`, `受体激动` |
| 给药/制剂特点 | `口服`, `注射剂`, `缓释`, `靶向制剂` |
| 患者人群 | `儿童`, `妊娠期`, `老年人`, `肝损`, `肾损` |
| 临床场景 | `急救`, `慢病管理`, `围手术期` |
| 药学判断类 | `治疗窗窄`, `个体化用药`, `药物监测` |

**规则：**
- 全部小写或中文，不用英文缩写
- 来自正文高频关键词，优先选有检索价值的
- 不与 `field`、`category`、`essence` 重复

### summary — 摘要

| 子字段 | 要求 | 与正文的关系 |
|---|---|---|
| `short` | 一句话定义，≤40 字，纯陈述句，不含"本节/本章/本文" | 等价于袖珍定义，独立于正文 |
| `full` | `short` 的拓展完善版本，≤100 字，交代背景/意义/临床价值，可含专业术语 | 正文"本体"问的素材库，不等于正文全文 |

**示例：**

```yaml
# 概念类
summary:
  short: 首过效应是指口服药物经胃肠道吸收后，在进入体循环前被肝脏代谢使药量减少的现象。
  full: 口服药物经胃肠道吸收后须先经门静脉进入肝脏，部分药物在首次通过时被大量代谢，导致进入体循环的药量减少，直接影响口服药物的生物利用度。

# 药物类
summary:
  short: 美托洛尔是一种选择性β₁受体阻滞剂，主要用于高血压、心绞痛和心衰的治疗。
  full: 美托洛尔属于心血管药物中的β受体阻滞剂类，对β₁受体有较高选择性，阻断心脏β₁受体可减慢心率、降低心肌耗氧量。缓释制剂可实现每日一次平稳降压。

# 入口类
summary:
  short: 本章介绍抗癫痫发作药物的分类、药理机制、临床应用及用药监护要点。
  full: 抗癫痫发作药物是药学专业知识二第一章的核心章节，覆盖传统与新型抗癫痫药的作用机制、适应证、药代动力学特征及特殊人群用药监护。
```

### edges_out — 关系边

共 11 种，分为四组，含义从强到弱：

**总览：**

```
A. 结构与组成
   ├── has          组成（has）：整体 - 部分构成关系（整体→部分）
   └── isa          归类（isa）：实例 - 概念从属关系（具体→抽象）

B. 药理机制
   ├── activates    激动受体/通路
   ├── inhibits     抑制/拮抗受体/酶/通路
   ├── metabolizes  经代谢酶转化（可双向）
   └── mechanism    一般作用机制

C. 临床关联
   ├── treats       治疗适应证
   ├── causes       引起不良反应
   ├── interacts    药物相互作用（可双向）
   └── contraindicates 禁忌（可双向）

D. 学习路径
   └── prerequisite 认知前置（不懂A就学不懂B）
```

**A. 结构与组成**

| 值 | 含义 | 方向 | 示例 |
|---|---|---|---|
| `has` | 物理组成（整体含部分） | 制剂 → 辅料 | 布洛芬缓释胶囊 → 羟丙甲纤维素 |
| `isa` | 是个体/实例（具体归抽象） | 具体药 → 药理分类 | 美托洛尔 → β受体阻滞剂 |

**B. 药理机制**

| 值 | 含义 | 方向 | 示例 |
|---|---|---|---|
| `activates` | 激动受体/通路 | 药 → 受体/通路 | 地西泮 → GABA_A受体 |
| `inhibits` | 抑制/拮抗 | 药 → 酶/受体/通路 | 奥美拉唑 → H⁺/K⁺-ATP酶 |
| `metabolizes` | 经代谢酶转化 | 药 ↔ 代谢酶（可双向） | 可待因 ↔ CYP2D6 |
| `mechanism` | 一般作用机制 | 药 → 靶点/过程 | 阿司匹林 → 环氧合酶抑制 |

**C. 临床关联**

| 值 | 含义 | 方向 | 示例 |
|---|---|---|---|
| `treats` | 治疗适应证 | 药 → 疾病/症状 | 丙戊酸钠 → 癫痫 |
| `causes` | 引起不良反应 | 药/因 → 果 | 阿司匹林 → 胃溃疡 |
| `interacts` | 药物相互作用 | 药 ↔ 药（可双向） | 华法林 ↔ 阿司匹林 |
| `contraindicates` | 禁忌 | 药 → 疾病/状态/人群 | 沙利度胺 → 妊娠 |

**D. 学习路径**

| 值 | 含义 | 方向 | 示例 |
|---|---|---|---|
| `prerequisite` | 认知前置（不懂A就学不懂B） | 概念A → 概念B | 首过效应 → 生物利用度 |

**填写规则：**

1. 先确认节点本质：结构/组成类用 `has`；药理类用 `activates`/`inhibits`/`mechanism`/`metabolizes`；临床类用 `treats`/`causes`/`interacts`/`contraindicates`；学习类用 `prerequisite`
2. 尽量选最具体的边类型，不用泛化的万能边
3. `metabolizes`、`interacts`、`contraindicates` 可以是双向边
4. 不要为"有边而有边"；`target` 必须指向仓库里真实存在的节点 id

**各类节点最低要求：**

| 节点类型 | 最低边数 | 重点边类型 |
|---|---|---|
| 篇/章/节入口 | 1 条 `has` + 1–2 条下游关联 | `has`、`prerequisite` |
| 概念类 | 1 条归属边 + 关键上下游 | `isa`、`prerequisite` |
| 具体药物类 | 1 条归属边 + 机制/适应证/禁忌 | `activates`、`inhibits`、`mechanism`、`treats`、`causes`、`contraindicates` |
| 制剂/工艺类 | 1 条结构归属边 + 组成/实例关联 | `has`、`isa` |
| 疾病类 | 1 条归属边 + 治疗药/病因 | `treats`、`causes` |

**完成标准：** frontmatter 符合当前 schema + 正文五问成立 + `edges_out` 不为空且反映正文关键关系 + 所有 `target` 能在现有节点 id 中找到 + 边类型从当前 11 种中选取，不出现其他值。

## 二、正文规范（五问法）

正文目标不是逐点解释，而是帮用户**从点到线，再到面**：搞懂当前节点 → 搞懂它和上下游的关系 → 把关系收束成整体结构。

### 五个维度

**理念：** frontmatter 里每一个字段都有它存在的意义，正文就是把它们的值全部用大白话讲出来——字段是骨架，人话是血肉。

**原则：**
- 每一问的答案都能对应到至少一个 frontmatter 字段，不能凭空杜撰
- 五答各覆盖一个维度，互不重叠；任意去掉一答，读者就少知道一件事
- 第三答和第五答必须让读者感受到"这张网"：谁连着谁、为什么连、在哪条线上
- 不用英文原词，不用字段名，不用标签堆砌

**目的：** 读者读完五问，不查资料也能说清楚"这是什么、和谁有关、为什么学、学到什么阶段、在知识体系里排哪"。

每篇正文覆盖以下五个维度，维度固定，问句可根据节点类型选择具体写法：

| 维度 | 回答的是什么 | 对应字段 | 回答原则 |
|---|---|---|---|
| **本体** | 这个节点到底是什么 | `label + essence + summary.short` | 先定身份，说清是药/概念/机制/入口，不展开细节 |
| **意义** | 为什么要理解它 | `summary.full + tier` | 说清学习价值/理解价值/应用价值，不能只说"很重要" |
| **关联** | 它主要和什么有关 | `field + category + tags + edges_out` | 把 edges_out 翻译成人话，组织成有逻辑的关联网络，不要堆标签 |
| **过程** | 它在什么阶段出现 | `tier` + 流程位置 | 明确先后顺序（学习顺序/体内过程/临床流程），不要写成位置介绍 |
| **结构** | 它在整体框架里放哪里 | `location` | 交代书本层级和知识结构层级，说清是总入口/中间层/下位节点 |

### 如何选择每一问的具体写法

**判断顺序：** 先根据 `essence` 确定节点大类 → 再根据 `location` 判断上下文角色 → 最后在对应类里选最贴切的问法组合。

**本体问的选择：**

| 节点大类 | 本体问可选写法 |
|---|---|
| 具体药物 | "这是一种什么药？""它属于哪类药物？" |
| 基础概念 | "这个概念在讲什么？""它说的是什么？" |
| 机制/过程 | "这个机制/过程是怎么起作用的？""它描述的是哪一步？" |
| 疾病/状态 | "这是什么病/什么状态？""它有哪些主要表现？" |
| 制剂/工艺 | "这是什么制剂或给药途径？""它解决什么问题？" |
| 章节/篇入口 | "这一章/篇在讲什么？""它的核心主题是什么？" |

**意义问的选择：**

根据 `tier` 判断节点在整条线上的位置，选对应的价值表述方向：

| tier 位置 | 意义问可选写法 | 回答方向 |
|---|---|---|
| 底层基础 | "为什么它是最先要搞懂的？" | 强调它是后续内容的铺垫 |
| 中间层 | "理解它能帮我看懂什么？" | 强调它是连接上下游的枢纽 |
| 临床应用 | "它在用药实践中有什么用？" | 强调实际判断价值 |

**关联问的选择：**

根据 `edges_out` 的主要类型选择写法：

| 主要边类型 | 关联问可选写法 | 翻译方式 |
|---|---|---|
| `activates`/`inhibits` | "它主要影响哪些通路或受体？" | 边 → "它通过激动/抑制……起作用" |
| `treats`/`causes` | "它和哪些疾病或症状直接相关？" | 边 → "它治疗/会引起……" |
| `has`/`isa` | "它属于哪个更大的体系？" | 边 → "它属于……系统/分类" |
| `prerequisite` | "搞懂它之后能接着学什么？" | 边 → "它之后会用到……" |
| 混合型（多种边都有） | "它和哪几条线上的内容关系最密切？" | 把不同类型的边分别翻译成关系 |

**过程问的选择：**

根据 `tier` 判断它在什么"流"里：

| tier 对应的流 | 过程问可选写法 |
|---|---|
| 学习顺序流 | "学它之前要有什么基础？学完之后接着学什么？" |
| 体内过程流 | "它发生在吸收、分布、代谢还是排泄哪个环节？" |
| 临床流程流 | "它一般出现在诊断、治疗、监护哪个阶段？" |
| 疾病发展流 | "它是疾病早期、中期还是晚期的问题？" |

**结构问的选择：**

根据 `location` 判断它在书本里的层级：

| 层级 | 结构问可选写法 |
|---|---|
| 全书/总入口 | "它在整套药学知识里处在什么位置？" |
| 篇入口 | "它是药学专业知识几的第几篇，主要管哪块内容？" |
| 章节入口 | "它位于哪一章，和同篇其他章节是什么关系？" |
| 节/子节节点 | "它是第几章第几节的内容，和上下级主题怎么连？" |

**仍不确定时的保底问法：**

> - 本体："它是什么？"
> - 意义："为什么要理解它？"
> - 关联："它主要和什么有关？"
> - 过程："它通常在什么阶段出现？"
> - 结构："它在整套知识体系里放在哪里？"

### 生成正文的执行流程

1. 判断节点类型，选对应一行问法
2. 把字段转义为人话（不要原样搬字段值）：
   - `essence` → "具体药物 / 基础概念 / 机制 / 模块入口"
   - `field` → "属于哪个学科"
   - `category` → "更适合放在……系统里理解"
   - `tier` → "前面必须先建立的底层认识" / "已经更靠近临床使用"
   - `location` → "位于哪一篇哪一章，在整体结构里是什么位置"
   - `edges_out` → "它属于……" / "它通过……起作用" / "它常和……连起来看"
3. 五答各解决一个维度，互不重复；去掉任意一答，信息都应少一块
4. 把关系写出来：第三答和第五答必须有网络感，让用户知道"它和谁连着、为什么连、放在哪条线里"
5. 最后检查：没有直接复述英文枚举值；没有只罗列 tags；普通人读完后真的知道"它是什么、跟谁有关、放在哪了"

**写完后自检清单：**
1. 五问里每一问的答案都能说清楚对应的是 frontmatter 哪个字段？
2. 本体问有没有把"不是 X"也顺带交代清楚？
3. 意义问有没有只说"重要/常见/必考"？
4. 关联问有没有把 edges_out 翻译成完整句子而不是堆标签？
5. 过程问有没有写成"它在第二章第三节"这样的位置描述？
6. 结构问有没有交代在知识体系里的层级和上下游关系？
7. 五答合计是否在 180–320 字之间？

**正文：**

```markdown
# 首过效应

## 它在讲什么？
它是一个生物药剂学里的基础概念，讲的是药物口服后从肠道进入血液之前，会先被肝脏"截留"一部分、代谢掉一部分，真正进入全身循环的药量因此变少的现象。

## 为什么要理解它？
理解首过效应，才能明白为什么同一种药口服和注射的效果差这么多——口服药受肝脏代谢影响大，生物利用度往往远低于注射剂。这也是为什么有些药只能口服、有些只能注射，剂型选择背后其实由这个概念决定。

## 它主要和什么有关？
它和口服给药、肝脏代谢、生物利用度、门静脉循环这些内容紧紧绑在一起。往前，它是理解"生物利用度"的前提；往后，它直接影响药物剂型设计和给药方案调整，比如硝酸甘油就不口服而用舌下含服，就是绕开首过效应的例子。

## 它通常在什么阶段出现？
它出现在理解药物体内过程的早期，是学习吸收环节时最先建立的底层认识——先懂了这个，后面学生物利用度、剂型比较和给药途径选择才顺理成章。

## 它在整套知识框架里放在哪里？
它位于药学专业知识一生物药剂学部分、药物吸收这一节，属于生物药剂学的基础概念层，往上承接口服给药的基本前提，往下通向生物利用度和临床给药决策。
```

**字数：** 五答合计约 300 字。

## 三、批量写节点的推荐顺序

**一句话原则：先用样板定标准，再用主链建导航，最后回补通过校验但仍然"走不通"的薄弱节点。**

### 推荐顺序

1. **先做一条样板链条**：挑一条层级清楚、概念关联明显的小链（最好包含篇入口/章入口/节点/具体概念），跑通写法和边密度标准
2. **补直接上游入口**：节→章→篇→总入口，最快形成可导航主线
3. **补同链的直接下游兄弟节点**：已写`药物吸收`，就继续补`生物利用度`、`首过效应`，不跳到别处
4. **扩到相邻主链的篇/章入口**：一条链通后，再去补相邻模块入口（性价比最高）
5. **回补通过校验但关系偏薄的节点**：重点看总入口、篇/章入口、正文提到很多关联但边只有 0–1 条的节点

### 为什么这样做

- **先定样板**：统一正文口径和边密度，避免后面批量返工
- **先打主链**：图谱最早从"能看点"变成"能导航"
- **每轮小闭环**：每补完一小批就运行 `npm run validate`，扩写速度和结构可靠性并行

### 可以调整顺序的情况

- 某节点是很多边共同指向的关键枢纽 → 可优先补
- 某上游已经写好，下游某一簇明显断链 → 可先修那一簇
- 用户明确指定范围 → 以用户要求为准
