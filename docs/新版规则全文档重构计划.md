> 编写日期：2026/6/11
> 目标：将 content/ 下全部占位文档逐个重写到 `docs/frontmatter.md` 最新 schema
> 状态：准备重置，从头开始

---

## 核心原则

- **现有文件全部视为占位符**，最终每个文件都会按新版规则逐个重写
- **不依赖批量脚本**，每篇正文五问、每个 frontmatter 字段均手工逐文件重写
- `layer` 字段不符合新版语义（新版标注为"备选可选"）→ 删除
- `type` → `essence` 全量重命名
- `field` + `tier` 双缺失 → 每个文件重写时补全
- `category` 值按新版系统分组规则逐文件判断

### 统一图谱核心原则

> 2026/6/11 修订

- **全部 216 个节点必须在蓝图表格里**，每行对应一个节点，每行必须包含完整的 frontmatter 元数据
- **id 全局唯一且互不冲突**，所有 `edges_out.target` 只能指向表格内已存在的 id；验证方式：扫描所有 edges_out.target，确保每个 target 都在 id 列中有对应行
- **三册书是统一的药学知识图谱**，不是三个独立模块；edges_out 可以跨书册指向（如药学综合知识与技能的节点可以指向药学专业知识二的节点）
- **edges_out 的 target 必须在终局填充阶段填写**（当前阶段可先用 `target-id | type | reason` 格式占位，待目标节点 id 确定后回填）
- **edges_out 的 11 种合法 type**：`has / isa / activates / inhibits / metabolizes / mechanism / treats / causes / interacts / contraindicates / prerequisite`

## 重写标准速查

每重写一个文件，对照以下清单：

### frontmatter 必填字段

- [x] `id` — label 对应的英文（连字符连接），不用拼音
- [x] `label` — 显示名称，不含"第几篇/第几章"序号
- [x] `essence` — 节点本质（`notion`/`medication`/`illness`/`route`/`substance`/`process`/`module`/`section`）
- [x] `field` — 学科领域（`pharmaceutics`/`pharmacokinetics`/`medicinal_chemistry`/`pharmacology`/`toxicology`/`biopharmaceutics`/`clinical_pharmacy`/`pharmacy_service`）
- [x] `tier` — 自然分层级（`basic`/`drug`/`disease`/`management`/`service`/`legal`）
- [x] `location` — 书本层级（book 必填，chapter 必填，其余按实际层级写）
- [x] `tags` — 3–8 个，不与 field/essence 重复
- [x] `summary.short` — ≤40 字，纯陈述句
- [x] `summary.full` — ≤100 字，交代背景/意义/临床价值
- [x] `edges_out` — 不为空，每条含 `type`/`reason`，`target` 字段在终局统一填充阶段填写；`type` 必须在 11 种合法类型中选取

### edges_out 11 种合法类型

```
has / isa / activates / inhibits / metabolizes / mechanism / treats / causes / interacts / contraindicates / prerequisite
```

### 正文字数与格式

- [x] 五问维度完整（本体/意义/关联/过程/结构）
- [x] 第一问按 `essence` 类型选择对应写法
- [x] 五答合计 180–320 字，每答 1–3 句
- [x] 关联问把 edges_out 翻译成完整句子，不堆标签
- [x] 无字段名/英文原词直搬

### 需删除的旧字段

- [x] `layer:` 行（新版语义不同，备选可选，删除）
- [x] `type:` 行（已改名为 `essence`）
- [x] edges_out 中的非 11 种类型
